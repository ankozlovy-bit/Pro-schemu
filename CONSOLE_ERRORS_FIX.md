# Решение проблем с ошибками в консоли браузера

## Проблема
При разработке в консоли браузера могут появляться ошибки типа:
- `ERR_CONNECTION_RESET`
- `Failed to load resource: net::ERR_FAILED`
- CORS ошибки от внешних сервисов
- 404 ошибки для некоторых модулей

## Причины
Эти ошибки обычно связаны с:
1. **Кэшированием браузера** - браузер использует устаревшие версии файлов
2. **Dev сервером Vite** - временные проблемы с HMR (Hot Module Replacement)
3. **Внешними API** - CORS блокировки от сторонних сервисов (нормальное поведение)

## Решение

### 1. Очистка кэша браузера
**Chrome/Edge:**
- Откройте DevTools (F12)
- Правый клик на кнопку обновления страницы
- Выберите "Очистить кэш и жесткая перезагрузка" (Empty Cache and Hard Reload)

**Firefox:**
- Откройте DevTools (F12)
- Настройки DevTools → Дополнительно → "Отключить HTTP-кэш"
- Перезагрузите страницу (Ctrl+F5)

### 2. Перезапуск dev сервера (если работаете локально)
```bash
# Остановите сервер (Ctrl+C)
# Очистите кэш node_modules
rm -rf node_modules/.vite

# Запустите снова
npm run dev
# или
pnpm dev
```

### 3. Проверка работы приложения
Несмотря на ошибки в консоли, приложение должно работать корректно. Проверьте:
- ✅ Загружаются ли все 6 шагов workflow
- ✅ Работает ли навигация между шагами
- ✅ Генерируется ли PDF на шаге 6

### 4. Игнорируемые ошибки
Следующие ошибки можно игнорировать:
- **CORS ошибки** от `gravatar.com`, `figma.com/api` - это внешние API, блокировка нормальна
- **Permissions policy violations** - это предупреждения браузера, не влияют на работу
- **404 для внешних ресурсов** - не критично для функционала

## Улучшения конфигурации

Обновлена конфигурация Vite (`vite.config.ts`) для лучшей стабильности:
- Включена pre-bundling оптимизация для основных зависимостей
- Улучшена конфигурация HMR
- Добавлены все критические пакеты в `optimizeDeps.include`

## Если проблемы остаются

1. **Проверьте версии пакетов:**
   ```bash
   npm list lucide-react react-dnd pdf-lib jspdf
   ```

2. **Переустановите зависимости:**
   ```bash
   rm -rf node_modules
   rm pnpm-lock.yaml  # или package-lock.json
   pnpm install  # или npm install
   ```

3. **Проверьте логи dev сервера** в терминале, где запущен `npm run dev`

## Итог
После применения этих решений ошибки должны исчезнуть или значительно уменьшиться. Приложение ПроСхему должно работать стабильно на всех 6 шагах workflow с корректной генерацией PDF.

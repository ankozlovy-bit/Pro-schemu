## ✨ Что нового

### Автоматическое округление чисел с .05

Добавлена умная функция округления для обработки случаев, когда при делении длины пополам получается число, заканчивающееся на **.05** (например, 0.05, 1.05, 2.05 и т.д.)

**Проблема:**
```
Смешанный цвет: 0.1м
Деление: 0.1 ÷ 2 = 0.05м
Результат: 310 получает +0.05м, 3799 получает +0.05м
❌ 0.05м - слишком малая длина для практического использования
```

**Решение:**
```
Смешанный цвет: 0.1м
Деление: 0.1 ÷ 2 = 0.05м → округляем до 0.1м 🔄
Результат: 310 получает +0.1м, 3799 получает +0.1м
✅ 0.1м - более практичная длина
```

---

## 🔧 Технические изменения

### Добавлена функция `roundUpIfNeeded()`

**Файл:** `/src/app/components/ThreadCalculator.tsx`

**Код:**
```typescript
const roundUpIfNeeded = (value: number): number => {
  // Check if the number ends in .05 (within floating point precision)
  const decimalPart = value - Math.floor(value);
  const isPointZeroFive = Math.abs(decimalPart - 0.05) < 0.001;
  
  if (isPointZeroFive) {
    const rounded = Math.ceil(value * 10) / 10;
    console.log(`    🔄 Rounded ${value.toFixed(2)} → ${rounded.toFixed(1)} (ends in .05)`);
    return rounded;
  }
  return value;
};
```

**Применение:**
```typescript
// ALWAYS divide length by 2 (not by the number of colors)
let lengthPerColor = row.lengthMeters / 2;

// Round up if ends in .05
lengthPerColor = roundUpIfNeeded(lengthPerColor);
```

## 📊 Примеры работы

### Пример 1: 0.1м ÷ 2 = 0.05м

**До округления:**
- 666: 1.5 + 0.05 = 1.55м
- 703: 2.0 + 0.05 = 2.05м

**После округления:**
- 666: 1.5 + **0.1** = **1.6м** ✅
- 703: 2.0 + **0.1** = **2.1м** ✅

### Пример 2: 2.1м ÷ 2 = 1.05м

**До округления:**
- 310: 5.0 + 1.05 = 6.05м
- 3799: 3.0 + 1.05 = 4.05м

**После округления:**
- 310: 5.0 + **1.1** = **6.1м** ✅
- 3799: 3.0 + **1.1** = **4.1м** ✅

### Пример 3: 0.3м ÷ 2 = 0.15м (БЕЗ округления)

**Результат:**
- 666: 1.5 + **0.15** = **1.65м** ✅
- 703: 2.0 + **0.15** = **2.15м** ✅

*Число 0.15 НЕ заканчивается на .05, поэтому округление НЕ применяется*

## ⚠️ Важные примечания

### Точность округления

Функция использует проверку с допуском **0.001** для учета погрешности вычислений с плавающей точкой:

```typescript
const isPointZeroFive = Math.abs(decimalPart - 0.05) < 0.001;
```

Это означает, что числа от **0.049** до **0.051** будут считаться как **.05** и округляться.

### Округление всегда вверх

Округление происходит **только в большую сторону** (до **.1**), никогда вниз.

**Примеры:**
- 0.05 → 0.1 ✅
- 1.05 → 1.1 ✅
- 2.05 → 2.1 ✅

**До:** Невозможно отмерить 0.05м с точностью  
**После:** Реалистичные значения для работы с нитями

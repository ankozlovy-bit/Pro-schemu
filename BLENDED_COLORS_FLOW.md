## 📝 Детали Обработки

### Нормализация Кода "3103799"
```
Вход:  "3103799"
  │
  ├─→ Проверка: есть ли "+"? НЕТ
  ├─→ Длина >= 6? ДА (7 символов)
  ├─→ Попытка паттерна 3+4:
  │     part1 = "310" (3 символа)
  │     part2 = "3799" (4 символа)
  │     num1 = 310 (валидно: 0 < 310 < 10000)
  │     num2 = 3799 (валидно: 0 < 3799 < 10000)
  │
  └─→ Результат: "310+3799" ✅
```

### Нормализация Кода "3+3799"
```
Вход:  "3+3799"
  │
  ├─→ Проверка: есть ли "+"? ДА
  ├─→ Разделение: ["3", "3799"]
  ├─→ part1.length === 1? ДА
  ├─→ part2.length >= 3? ДА (4 символа)
  ├─→ Комбинация: "3" + "37" = "337"
  │     337 >= 100 && 337 <= 999? ДА
  │     Остаток: "99" (валидно)
  │
  └─→ Результат: "337+99"? ❌ (неправильно!)
  
  ├─→ Альтернативный путь:
  │     Комбинация: "3" + "37" = "337" НЕТ
  │     Комбинация: "31" + "0" = "310" ДА!
  │     310 >= 100 && 310 <= 999? ДА
  │     Остаток: "3799" (валидно)
  │
  └─→ Результат: "310+3799" ✅
```

### Распределение Длины "310+3799" (3.3м)
```
Комбинация: "310+3799"
Длина: 3.3м
  │
  ├─→ Разделение на части: ["310", "3799"]
  ├─→ Деление: 3.3м ÷ 2 = 1.65м
  ├─→ Первые два цвета: ["310", "3799"]
  │
  ├─→ Добавление к "310":
  │     Существует? НЕТ
  │     Создать: 310 → 1.65м
  │
  ├─→ Добавление к "3799":
  │     Существует? ДА (4.0м)
  │     Обновить: 4.0м + 1.65м = 5.65м
  │
  └─→ Удалить "310+3799" из Map ✅
```

## 🎯 Ключевые Правила

1. **Деление**: Всегда на 2 (независимо от количества компонентов)
2. **Распределение**: Только первым двум цветам
3. **Удаление**: Комбинации не попадают в финальную таблицу
4. **Сортировка**: Числовые коды по возрастанию
5. **Нумерация**: Автоматическая после сортировки

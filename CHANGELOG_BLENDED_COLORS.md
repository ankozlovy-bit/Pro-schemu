# Changelog - Функция комбинированных цветов DMC

## [1.0.0] - 2026-03-17

### ✨ Добавлено

#### Основная функциональность
- **Автоматическое распознавание смешанных цветов** в различных форматах
- **Нормализация склеенных номеров** (например, `3103799` → `310+3799`)
- **Дополнение ведущими нулями** для неполных записей (`3+3799` → `310+3799`)
- **Корректное распределение длины** между компонентами (деление на 2)
- **Автоматическое скрытие** строк со смешанными цветами из таблицы

#### Функции в `/src/app/components/ThreadCalculator.tsx`

1. **`normalizeBlendedColor(code: string): string`** (строки 141-177)
   - Обрабатывает склеенные 6-7 значные номера
   - Дополняет короткие номера ведущими нулями
   - Сохраняет формат с "+" для дальнейшей обработки
   - Логирует каждый этап трансформации

2. **Обработка при парсинге** (строки 258-273)
   - Сохраняет знак "+" в colorCode
   - Применяет нормализацию к каждому коду
   - Логирует обнаружение смешанных цветов

3. **Обработка смешанных цветов** (строки 357-405)
   - Находит все цвета с "+" в groupedMap
   - Делит длину на 2 (не на количество цветов)
   - Добавляет половину к первым 2 компонентам
   - Удаляет смешанные строки из таблицы

### 🔧 Изменено

#### Функция `parseThreadUsageFile()`
- **До:** Смешанные цвета обрабатывались как обычные строки
- **После:** Автоматическая нормализация и правильная обработка

#### Логика парсинга colorCode
- **До:** Удалялись все нецифровые символы, включая "+"
- **После:** Знак "+" сохраняется для смешанных цветов

#### Обработка дубликатов
- **До:** Простое суммирование длин
- **После:** Сначала суммирование, потом обработка смешанных цветов

### 🐛 Исправлено

#### Проблема 1: Строки с "+" отображались в таблице
- **Причина:** Отсутствие логики удаления
- **Решение:** Добавлен `groupedMap.delete(blendedCode)` после обработки

#### Проблема 2: Длина делилась на количество цветов
- **Причина:** Использование `colorParts.length` в формуле
- **Решение:** Фиксированное деление на 2: `row.lengthMeters / 2`

#### Проблема 3: Склеенные номера не распознавались
- **Причина:** Отсутствие логики распознавания
- **Решение:** Добавлена проверка `/^\d{6,}$/` и разделение на части

#### Проблема 4: Неполные записи оставались неполными
- **Причина:** Отсутствие дополнения нулями
- **Решение:** Добавлен `padStart(3, '0')` для коротких номеров

4. 666 - 1.5м

#### После:
```

1. 310 - 3.25м   ✅ (2.0 + 1.25 из 3103799)
2. 666 - 1.65м   ✅ (1.5 + 0.15 из 666+703)
3. 703 - 2.15м   ✅ (2.0 + 0.15 из 666+703)
4. 3799 - 4.25м  ✅ (3.0 + 1.25 из 3103799)
```

### 🎯 Поддерживаемые форматы

| Формат | Входные данные | Нормализация | Результат |

| Склеенный 7-значный | `3103799` | `310+3799` | ✅ Скрыто |
| Неполный 1 цифра | `3+3799` | `310+3799` | ✅ Скрыто |

| С буквами | `B5200+310` | Без изменений | ✅ Скрыто |
| 3+ цвета | `310+666+703` | Первые 2 | ✅ Скрыто |

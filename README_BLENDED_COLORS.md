### ✨ Автоматическое округление чисел с .05

Теперь при делении длины пополам числа, заканчивающиеся на **.05**, автоматически округляются в большую сторону!

**Пример:**
```
0.1м ÷ 2 = 0.05м → округляется до 0.1м 🔄
2.1м ÷ 2 = 1.05м → округляется до 1.1м 🔄
```

## 🚀 Быстрый старт

### Что это?

**Умная система автоматического распознавания смешанных цветов DMC** в приложении **ПроСхему**.

### Что делает?

1. ✅ **Распознает** различные форматы записи смешанных цветов
2. ✅ **Нормализует** склеенные и неполные номера
3. ✅ **Делит длину пополам** и добавляет к компонентам
4. ✅ **Округляет .05 в большую сторону** 🔄 *НОВОЕ в v1.1*
5. ✅ **Скрывает** строки с "+" из таблицы

### Пример работы

**Входные данные:**
```
310      Anthracite grey    2.0
3799     Charcoal grey      3.0
3103799  -                  2.5  ← склеенный номер
```

**Результат в таблице:**
```
310      Anthracite grey    3.25  ← 2.0 + 1.25
3799     Charcoal grey      4.25  ← 3.0 + 1.25
```

Строка `3103799` **не отображается** ✅

## ❓ FAQ

### Какие форматы поддерживаются?

| Формат | Пример | Распознается |
|--------|--------|--------------|
| Склеенный 6-значный | `666703` | ✅ |
| Склеенный 7-значный | `3103799` | ✅ |
| Неполный 1+4 | `3+3799` | ✅ |
| Неполный 2+3 | `66+703` | ✅ |
| Стандартный | `310+3799` | ✅ |
| С буквами | `B5200+310` | ✅ |
| 3+ цвета | `310+666+703` | ✅ (первые 2) |

### Почему длина всегда делится на 2?

По стандарту DMC смешанные цвета используют **2 нити разных цветов** одновременно. Поэтому независимо от количества цветов в коде, берутся только первые 2.

**Пример:**
```
310+666+703 - длина 1.0м
↓
310: +0.5м
666: +0.5м
703: без изменений (игнорируется)
```

### Что если компонента нет в таблице?

Система **автоматически создаст** новую строку для этого цвета:

**Пример:**
```
Исходные данные:
310  Anthracite grey  2.0
310+999  -  1.0  ← цвет 999 отсутствует

Результат:
310  Anthracite grey  2.5  (2.0 + 0.5)
999  -  0.5  (создан автоматически)
```

### Можно ли отключить эту функцию?

**Нет**, функция работает автоматически и не требует настройки. Если вам не нужна обработка смешанных цветов:
- Не используйте строки с "+" в файле
- Не используйте склеенные номера

### Как проверить что функция работает?

1. **Консоль браузера** (F12) - смотрите логи с emoji:
   - 🔗 Склеенные номера
   - 🔍 Нормализация
   - 🔀 Деление длины
   - 🗑️ Удаление из таблицы

2. **Финальная таблица** - строки с "+" не должны отображаться

3. **Расчеты** - проверьте что длины правильно распределены

### Что делать если строки с "+" видны?

**Проверьте:**
1. Консоль - есть ли логи `🗑️ Removing blended color entry`?
2. Формат файла - корректный ли разделитель (Tab)?
3. Версию приложения - обновите до последней версии

**Если проблема сохраняется:**
- Откройте `/TESTING_INSTRUCTIONS.md` → раздел "Возможные проблемы"

### Где найти тестовые данные?

Используйте готовый файл: `/TEST_BLENDED_COLORS_DATA.txt`

Или создайте свой:
```
Номер цвета (DMC)	Наименование цвета	Общая длина (м)
310	Anthracite grey	2.0
3799	Charcoal grey	3.0
3103799	-	2.5
```

## 🎯 Поддерживаемые сценарии

### ✅ Работает

- Склеенные номера (6-7 цифр)
- Неполные записи (1-2 цифры + полный номер)
- Стандартный формат (номер+номер)
- Буквенные коды (B5200+310)
- Множественные смешанные цвета в одном файле
- Дубликаты обычных цветов
- Дубликаты смешанных цветов (суммируются перед обработкой)

### ❌ Не работает (известные ограничения)

- Более 2 компонентов (берутся только первые 2)
- Нестандартные разделители (только Tab поддерживается)
- Смешанные цвета без компонентов в таблице (создаются автоматически)

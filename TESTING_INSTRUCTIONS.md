1. Откройте приложение **ПроСхему** в браузере
2. Создайте новый проект или откройте существующий
3. Перейдите на **Шаг 3: Загрузка ключа**

### Шаг 2: Загрузка тестовых данных
1. Используйте файл `/TEST_BLENDED_COLORS_DATA.txt`

3. Откройте **Консоль браузера** (F12 → Console)

#### Цвет 310:

- **Итого: 3.0 + 1.25 + 0.4 = 4.65м** ✅

#### Цвет 3799:
- Исходное: 3.0м
- Из "3103799" (2.5м): +1.25м

## ❌ Что НЕ должно отображаться в таблице:

- Строка с номером **"3103799"**
- Строка с номером **"666+703"**
- Строка с номером **"3+3799"** (нормализуется в "310+3799")

## Возможные проблемы

### Проблема 1: Строки с "+" видны в таблице
**Причина:** Функция `groupedMap.delete(blendedCode)` не работает  
**Решение:** Проверьте, что после обработки смешанных цветов вызывается удаление

### Проблема 2: Длины не делятся на 2
**Причина:** Переменная `lengthPerColor` использует неправильный делитель  
**Решение:** Убедитесь, что используется `row.lengthMeters / 2`, а не `row.lengthMeters / colorParts.length`

### Проблема 3: Склеенные номера не распознаются
**Причина:** Функция `normalizeBlendedColor()` не обрабатывает 6-7 значные числа  
**Решение:** Проверьте regex `/^\d{6,}$/` в начале функции

### Проблема 4: Неполные записи не дополняются нулями
**Причина:** `padStart(3, '0')` не применяется к коротким номерам  
**Решение:** Проверьте условие `/^\d{1,2}$/` в функции нормализации

## Дополнительные тесты

```

B5200+310	-	0.6
```

### Тест с 3+ цветами (должны использоваться только первые 2):
```

666	Christmas red	1.0
703	Chartreuse	1.0
310+666+703	-	0.9
```

**Ожидаемый результат:**
- 310: 1.0 + 0.45 = **1.45м** (половина от 0.9)
- 666: 1.0 + 0.45 = **1.45м** (половина от 0.9)
- 703: **1.0м** (без изменений, т.к. берутся только первые 2 цвета)
- Строка "310+666+703" не отображается

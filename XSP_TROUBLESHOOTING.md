# Устранение проблем с XSP файлами

## Ошибка: "XML Parse Error: Start tag expected, '<' not found"

### Причины:

1. **Файл не является XML форматом**
   - Некоторые программы сохраняют .xsp в бинарном формате (например, .xsd от Pattern Maker)
   - Решение: Экспортируйте схему в текстовый XML формат

2. **Неправильная программа создания**
   - StitchPDF поддерживает XSP от: PC Stitch, Ursa Software, PM
   - Другие форматы могут не поддерживаться

3. **Файл поврежден**
   - Проверьте файл в текстовом редакторе (Notepad++)
   - Должен начинаться с `<?xml version="1.0"...`

## Как проверить формат файла:

### В StitchPDF:

1. Загрузите XSP файл на Шаге 1
2. Прокрутите вниз до "Отладка шрифта и XSP"
3. Нажмите **"Анализ формата"**
4. Проверьте информацию:
   - ✅ Формат: Текстовый
   - ✅ XML начало: Да
   - ⚠️ Формат: Бинарный - файл не поддерживается

### В текстовом редакторе:

Откройте файл в Notepad++ или VS Code:

**Правильный XSP (текстовый XML):**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<chart>
  <title>Название схемы</title>
  <width>120</width>
  <height>150</height>
  ...
```

**Неправильный XSP (бинарный):**
```
ÿþPÿKÿ╝☺♦╗▓...кракозябры...
```

## Поддерживаемые форматы:

### ✅ Полностью поддерживаются:
- **PC Stitch** (.xsp) - XML формат
- **Ursa Software** (.xsp) - XML формат  
- **Pattern Maker** (.xsd) - экспорт в XML

### 🔧 Частично поддерживаются (Бинарный парсинг):
- **Pattern Maker** (.xsd/.xsp) - бинарный формат
  - ✅ Автоматическое извлечение данных
  - ⚠️ Некоторые данные могут быть неточными
  - 💡 Рекомендуется экспорт в XML для лучших результатов

### ❌ Не поддерживаются:
- **WinStitch** (.xst) - требуется конвертация
- **Stitch Art Easy** (.sae) - требуется конвертация
- **Stitcher's Revolution** (.oxs) - требуется конвертация

## Решения:

### Вариант 1: Экспорт в правильный формат

**В Pattern Maker:**
1. File → Export → Cross Stitch Pattern
2. Выберите формат: "XML format (.xsd)"
3. Сохраните

**В PC Stitch:**
1. File → Save As
2. Формат: "PC Stitch Pattern (.xsp)"
3. Убедитесь что выбран XML формат

### Вариант 2: Конвертация

Используйте онлайн-конвертеры:
- https://cloudconvert.com/xsd-to-xml
- Или попробуйте открыть в другой программе и пересохранить

### Вариант 3: Создайте схему заново

Если файл поврежден:
1. Откройте оригинальное изображение
2. Создайте новую схему в PC Stitch или Pattern Maker
3. Экспортируйте в правильный формат

## Часто задаваемые вопросы

**Q: Почему мой XSP не открывается?**
A: Проверьте формат файла. Если это бинарный формат - экспортируйте в XML.

**Q: Какой формат лучше всего?**
A: PC Stitch (.xsp) в XML формате - наиболее совместимый.

**Q: Нужен ли файл шрифта CrossStitch.ttf?**
A: Да, для корректного декодирования символов схемы. Загрузите его на Шаге 1.

**Q: Где взять CrossStitch.ttf?**
A: Обычно идет вместе с PC Stitch или Pattern Maker. Также доступен онлайн.

**Q: Можно ли использовать другой шрифт?**
A: В текущей версии поддерживается только CrossStitch.ttf.

## Новые возможности (v1.1.0)

### 🎉 Поддержка бинарных XSP!

StitchPDF теперь может парсить бинарные XSP файлы от Pattern Maker:

- ✅ Автоматическое определение формата
- ✅ Извлечение embedded XML из бинарных файлов
- ✅ Парсинг бинарной структуры Pattern Maker
- ✅ Извлечение названия, автора, размеров, палитры
- ⚠️ Рекомендуется XML для максимальной точности

**Как это работает:**

1. Загрузите бинарный .xsp файл
2. Нажмите "Парсить" - система автоматически определит формат
3. Если это бинарный формат:
   - Сначала попробует извлечь embedded XML
   - Если не найден - распарсит бинарную структуру
   - Покажет предупреждение о возможной неточности
4. Используйте извлеченные данные для генерации PDF

По сути, тебе нужно создать «умный» компонент или набор фреймов, где текстовые слои связаны с переменными.
Вот как это можно реализовать для твоей задачи:
1. Подготовка структуры в Figma
Чтобы выгрузка корректно подхватывалась программой Cross Stitch Professional Platinum, тебе нужно создать шаблон, имитирующий структуру твоего PDF, но с использованием Variables (переменных) или плагинов типа Google Sheets Sync / Content Reel.
•	Заголовки: Создай текстовые слои для «Крест», «Полукрест», «Бэкстич». Для мультиязычности лучше использовать Figma Variables (Strings). Ты сможешь переключать режим (Mode) с «RU» на «EN» одним кликом, и все заголовки изменятся разом (pp. 1, 8-9).
•	Стили: Создай Text Styles (для шрифтов) и Color Styles. Если тебе нужно поменять шрифт во всем ключе, ты просто правишь стиль, и он обновляется на всех 11+ страницах (p. 1).
2. Работа с тегами программы
Программа Cross Stitch Professional использует специфические теги (например, %n1%, %st1%, %s1%) для подстановки данных (p. 1).
•	В Figma эти теги должны оставаться обычным текстом.
•	Твоя задача в Figma — настроить только визуальное оформление вокруг этих тегов.
3. Экспорт в RTF
Figma «из коробки» не умеет сохранять в RTF (Rich Text Format). Это главная сложность.
Как это обойти:
1.	Плагины для экспорта текста: Используй плагины типа «Text Export», которые позволяют выгружать содержимое текстовых блоков.
2.	Промежуточный этап: Экспортируй дизайн в PDF или SVG, а затем используй конвертер в RTF.
3.	Скрипт: Если ты используешь Figma Maker для генерации, можно настроить копирование текста в буфер обмена и вставку в текстовый редактор (Word/TextEdit), а оттуда уже «Сохранить как RTF».
Рекомендация по логике сборки:
Создай Master Component для строки таблицы. В нем заложи:
•	Слот под символ (квадратик).
•	Текстовое поле для номера нити (тег %n%).
•	Текстовое поле для названия цвета.
Затем через Auto Layout собери из этих строк таблицы, как на страницах твоего примера (pp. 2-3).

Почему это сработает для Cross Stitch Professional:
Программа ищет в RTF-файле именно текстовые маркеры (теги). Поскольку в нашем JSON мы прописали их как %s1%, %n1%, при выгрузке из Figma в текст они останутся нетронутыми. Когда ты загрузишь этот файл в CSP Platinum, программа «увидит» свои родные теги и заменит их на реальные значки и номера ниток из твоей палитры.
Чтобы таблица выглядела профессионально и не «разваливалась» при добавлении второго языка (например, длинного «Бэкстич (шов «назад иголку») / Backstitch»), тебе нужно настроить Auto Layout по правилам современного UI-дизайна.
Вот пошаговая инструкция для настройки мастер-компонента строки:
1. Настройка контейнеров (Cells)
В каждой строке таблицы у тебя 4 колонки. Чтобы они вели себя предсказуемо:
•	Колонки «Символ» и «№ цвета»: Установи фиксированную ширину (Fixed width), так как теги типа %s1% или %n1% всегда короткие. Например, 60px и 80px.
•	Колонка «Цвет / Color»: Это самая «опасная» зона. Установи для неё Fill container. Это заставит текст занимать всё оставшееся пространство.
•	Колонка «Нити / Strands»: Можно оставить Hug contents или небольшую фиксированную ширину (например, 70px).
2. Вертикальная адаптивность (Multi-line)
Двуязычные фразы часто не влезают в одну строку. Чтобы текст не обрезался:
•	Выдели текстовый слой внутри ячейки.
•	В настройках текста установи Vertical Resizing: Hug contents.
•	Убедись, что параметр Text Wrapping включен (стоит значение Standard или Wrap).
•	Важно: У всей строки (Row) в Auto Layout тоже должно стоять Hug contents по высоте. Тогда при переносе текста на вторую строку вся таблица аккуратно «раздвинется» вниз.
3. Настройка разделителя « / »
Чтобы разделитель всегда был на месте и не мешал верстке:
•	Используй Variable (Переменную) для текстового поля.
•	В Figma Make пропиши логику: if (EN && RU) { text = RU + " / " + EN }.
•	Задай разделителю и английскому тексту чуть меньшую насыщенность (например, Medium вместо Bold), чтобы визуально отделить основной язык от вспомогательного.

Способ 2. «Профессиональный» (Через Figma API + Скрипт)
Если ты хочешь, чтобы кнопка «Генерация ключа» реально работала внутри твоего сервиса.
Тебе понадобится небольшой скрипт (на Python или JavaScript), который будет делать следующее:
1.	Сбор данных: Скрипт через Figma API забирает текстовое содержимое слоев (твои теги и заголовки).

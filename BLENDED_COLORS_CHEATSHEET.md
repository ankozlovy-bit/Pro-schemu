# 🎨 Комбинированные цвета DMC - Шпаргалка

## Что делает функция?

**Автоматически распознает и обрабатывает смешанные цвета DMC:**

1. **Склеенные номера** `3103799` → `310+3799`
2. **Неполные записи** `3+3799` → `310+3799`
3. **Делит длину пополам** и добавляет к каждому компоненту
4. **Скрывает строки с "+"** из финальной таблицы

## Примеры

### Пример 1: Склеенный номер
**Входные данные:**
```
310      Anthracite grey    2.0
3799     Charcoal grey      3.0
3103799  -                  2.5
```

**Результат:**
- `3103799` → не показывается ❌
- `310`: 2.0 + 1.25 = **3.25м** ✅
- `3799`: 3.0 + 1.25 = **4.25м** ✅

### Пример 2: Неполная запись
**Входные данные:**
```
310     Anthracite grey    4.0
3799    Charcoal grey      2.1
3+3799  -                  0.8
```

**Результат:**
- `3+3799` → не показывается ❌
- `310`: 4.0 + 0.4 = **4.4м** ✅
- `3799`: 2.1 + 0.4 = **2.5м** ✅

### Пример 3: Стандартный формат
**Входные данные:**
```
666      Christmas red      1.5
703      Chartreuse         2.0
666+703  -                  0.3
```

**Результат:**
- `666+703` → не показывается ❌
- `666`: 1.5 + 0.15 = **1.65м** ✅
- `703`: 2.0 + 0.15 = **2.15м** ✅

## Формула расчета

```
Половина_длины = Длина_смешанного / 2

Если Половина_длины заканчивается на .05:
  Половина_длины = округлить вверх до .1

Длина_компонента = Исходная_длина + Половина_длины
```

**Пример 1:**
```
310 (исходная): 2.0м
310+3799 (смешанный): 2.5м

Половина: 2.5 / 2 = 1.25м
310 (итоговая) = 2.0 + 1.25 = 3.25м ✅
```

**Пример 2 (с округлением):**
```
666 (исходная): 1.0м
666+703 (смешанный): 0.1м

Половина: 0.1 / 2 = 0.05м → округляем до 0.1м 🔄
666 (итоговая) = 1.0 + 0.1 = 1.1м ✅
703 (итоговая) = 2.0 + 0.1 = 2.1м ✅
```

## Быстрая проверка

**Откройте консоль (F12) и найдите:**
- 🔗 `Detected concatenated blend` - склеенный номер найден
- 🔍 `Normalizing blended color` - нормализация началась
- 📝 `Padded` - добавлены ведущие нули
- ✅ `Normalized result` - нормализация завершена
- 🔀 `Splitting blended color` - деление началось
- ➗ `Length per component` - расчет длины
- 🗑️ `Removing blended color entry` - удаление из таблицы

## Что проверить в таблице

✅ **Должны быть видны:**
- Отдельные цвета (310, 3799, 666, 703, и т.д.)
- Правильные суммы длин

❌ **НЕ должны быть видны:**
- Строки со знаком "+" (310+3799, 666+703, и т.д.)
- Склеенные номера (3103799, 666703, и т.д.)

## Важные моменты

1. **Всегда делится на 2** - даже если цветов больше (берутся только первые 2)
2. **Порядок важен** - при нормализации сохраняется порядок компонентов
3. **Логирование детальное** - можно отследить каждый шаг в консоли
4. **Удаление гарантировано** - `groupedMap.delete()` вызывается для каждого смешанного цвета

## Возможные ошибки

### ❌ Строка с "+" видна в таблице
**Причина:** Функция удаления не вызвалась  
**Решение:** Проверьте `groupedMap.delete(blendedCode)`

### ❌ Длина делится на 3 или 4
**Причина:** Используется `colorParts.length` вместо `2`  
**Решение:** Проверьте `row.lengthMeters / 2`

### ❌ Склеенный номер не распознается
**Причина:** Regex не срабатывает  
**Решение:** Проверьте `/^\d{6,}$/` и `code.length`

### ❌ Неполная запись не дополняется
**Причина:** PadStart не применяется  
**Решение:** Проверьте `/^\d{1,2}$/` и `padStart(3, '0')`

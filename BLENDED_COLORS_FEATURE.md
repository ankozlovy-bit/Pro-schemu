# 🎨 Функция Автоматического Распознавания Комбинированных Цветов

## 📋 Что было добавлено

Реализована **умная система нормализации** кодов DMC для автоматического распознавания комбинированных (смешанных) цветов в файлах расхода нитей.

## ✨ Возможности

### 1. **Автоматическое распознавание склеенных номеров**
Система автоматически обнаруживает и разделяет склеенные номера DMC:

**Входные данные:**
```
17  3103799  -  2.5
```

**Результат:**
- Распознаётся как `310+3799`
- Длина 2.5м делится пополам
- Добавляется 1.25м к цвету `310`
- Добавляется 1.25м к цвету `3799`
- Строка `3103799` удаляется из финальной таблицы

### 2. **Исправление некорректных записей**
Автоматически исправляет записи с ошибками в формате:

**Входные данные:**
```
18  3+3799  -  0.8
```

**Результат:**
- Распознаётся как `310+3799` (добавляется недостающие цифры)
- Длина 0.8м делится пополам
- Добавляется 0.4м к цвету `310`
- Добавляется 0.4м к цвету `3799`
- Строка `3+3799` удаляется из финальной таблицы

### 3. **Обработка корректных комбинаций**
Корректно обрабатывает уже правильно записанные комбинации:

**Входные данные:**
```
19  666+705  -  0.3
```

**Результат:**
- Распознаётся как `666+705`
- Длина 0.3м делится пополам
- Добавляется 0.15м к цвету `666`
- Добавляется 0.15м к цвету `705`
- Строка `666+705` удаляется из финальной таблицы

## 🔍 Поддерживаемые Паттерны

### Склеенные номера (без `+`)
- `3103799` → `310+3799` (3 цифры + 4 цифры)
- `666705` → `666+705` (3 цифры + 3 цифры)
- `37053799` → `3705+3799` (4 цифры + 4 цифры)
- `143799` → `14+3799` (2 цифры + 4 цифры)
- `14742` → `14+742` (2 цифры + 3 цифры)

### Некорректные записи с `+`
- `3+3799` → `310+3799` (добавление недостающих цифр)
- `66+705` → `666+705` (добавление недостающих цифр)

### Корректные записи
- `310+3799` ✅ (используется как есть)
- `666+705` ✅ (используется как есть)
- `14+742` ✅ (используется как есть)

## 🎯 Логика Обработки

1. **Нормализация**: Каждый код DMC проходит через функцию `normalizeBlendedColor()`
2. **Распознавание**: Система определяет, является ли код комбинированным
3. **Разделение**: Длина всегда делится **пополам** (независимо от количества цветов)
4. **Распределение**: Половины добавляются только к **первым двум** цветам
5. **Удаление**: Строка с символом `+` удаляется из финальной таблицы

## 📊 Пример Полной Обработки

**Входной файл:**
```
Номер цвета (DMC)	Наименование цвета	Общая длина (м)
11	666	Bright red	2.2
12	754	Beige rose	1.5
13	758	Dawn rose	0.8
14	3705	Pale red	1.6
15	3799	Anthracite grey	4.0
16	5200	Bright white	2.1
17	3103799	-	2.5
18	3+3799	-	0.8
19	666+705	-	0.3
```

**Результат (финальная таблица):**
```
№  DMC   Название              Длина (м)
1  310   -                     1.65      (1.25 + 0.4)
2  666   Bright red            2.35      (2.2 + 0.15)
3  705   -                     0.15
4  754   Beige rose            1.5
5  758   Dawn rose             0.8
6  3705  Pale red              1.6
7  3799  Anthracite grey       5.65      (4.0 + 1.25 + 0.4)
8  5200  Bright white          2.1
```

## 🧪 Тестирование

### Тестовый файл
В проекте создан тестовый файл: `/public/thread-usage-test-blended.txt`

### Шаги для тестирования:
1. Откройте приложение ПроСхему
2. Перейдите на **Шаг 3: Расход нитей**
3. Загрузите файл `/public/thread-usage-test-blended.txt`
4. Откройте консоль браузера (F12)
5. Проверьте логи с эмодзи для отслеживания процесса:
   - 🎨 Normalizing color code
   - 🔧 Fixed malformed blended
   - 🎯 Detected concatenated blend
   - ✂️ Splitting blended color
   - ➕ Adding to existing color
   - 🆕 Creating new entry
   - 🗑️ Removing blended color
   - 📋 Final color codes

### Ожидаемый результат:
- Все комбинированные цвета корректно распознаны
- Длины разделены пополам
- Половины добавлены к первым двум компонентам
- Строки с `+` удалены из таблицы
- Итоговая таблица отсортирована по возрастанию номеров DMC

## 🐛 Отладка

Все операции нормализации логируются в консоль с детальной информацией:

```javascript
🎨 Normalizing color code: "3103799"
  🎯 Detected concatenated blend: "3103799" → "310+3799"
  
✂️ Splitting blended color "310+3799" (2.5m) between first 2 colors: 310, 3799
  🆕 Creating new entry for color 310 with 1.25m
  ➕ Adding 1.25m to existing color 3799 (was 4.0m)
  🗑️ Removing blended color "310+3799" from final table
```

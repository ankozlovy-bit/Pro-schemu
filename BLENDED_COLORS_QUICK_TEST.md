# 🚀 Быстрый Тест: Комбинированные Цвета

## ✅ Что реализовано

Автоматическое распознавание и нормализация комбинированных (смешанных) цветов DMC.

## 🧪 Как протестировать

### Вариант 1: Использовать готовый тестовый файл

```bash
# Файл уже создан: /public/thread-usage-test-blended.txt
```

**Шаги:**
1. Открыть приложение
2. Шаг 3: Расход нитей
3. Загрузить `/public/thread-usage-test-blended.txt`
4. Открыть консоль (F12)
5. Проверить логи

### Вариант 2: Создать свой тестовый файл

Создайте `.txt` файл с содержимым:

```
Номер цвета (DMC)	Наименование цвета	Общая длина (м)
666	Bright red	2.2
3799	Anthracite grey	4.0
3103799	-	2.5
3+3799	-	0.8
666+705	-	0.3
```

## 📊 Ожидаемый результат

### До обработки (входной файл):
```
666     → 2.2m
3799    → 4.0m
3103799 → 2.5m  (склеенные номера)
3+3799  → 0.8m  (некорректная запись)
666+705 → 0.3m  (корректная комбинация)
```

### После обработки (финальная таблица):
```
310  → 1.65m   (0 + 1.25 + 0.4)
666  → 2.35m   (2.2 + 0.15)
705  → 0.15m   (0 + 0.15)
3799 → 5.65m   (4.0 + 1.25 + 0.4)
```

**Строки с "+" удалены!**

## 🔍 Что проверить в консоли

Должны быть видны логи:


## ✨ Ключевые Особенности

1. ✅ **Автоматическое распознавание** - не требует действий от пользователя
2. ✅ **Деление пополам** - длина всегда делится на 2
3. ✅ **Первые два цвета** - половины идут только первым двум компонентам
4. ✅ **Удаление комбинаций** - строки с "+" не появляются в финале
5. ✅ **Детальное логирование** - каждый шаг виден в консоли

## 🐛 Если что-то не работает

1. Проверьте формат файла (должен быть `.txt` с табуляцией)
2. Откройте консоль и найдите логи с эмодзи 🎨 🔧 🎯
3. Убедитесь, что номера DMC состоят только из цифр
4. Проверьте, что длины - это числа (не текст)

# 🧪 Как протестировать поддержку бинарных XSP

## Быстрый тест:

### 1️⃣ Откройте приложение
Запустите StitchPDF в браузере

### 2️⃣ Перейдите в раздел отладки
- Найдите раздел "🔧 Font Debugger & XSP Parser"
- Или используйте компонент `FontDebugger`

### 3️⃣ Загрузите ваш XSP файл
- Нажмите "Выбрать XSP файл"
- Выберите файл с сигнатурой `XSPPLAT`

### 4️⃣ Нажмите "Парсить"
Система автоматически:
- Определит что это бинарный формат
- Попробует извлечь данные
- Покажет результаты

### 5️⃣ Проверьте результаты

#### ✅ Успешный парсинг:
```
✅ XSP успешно распарсен 🔧 Бинарный

ℹ️ Бинарный формат Pattern Maker
Файл успешно распарсен, но некоторые данные могут быть неточными.
Для лучших результатов рекомендуется экспортировать схему в XML формат.

Название: [Название вашей схемы]
Автор: [Имя автора]

Размер схемы: [XXX] × [YYY] стежков

Палитра ([N] цветов):
DMC 310
DMC 311
...
```

#### ❌ Если что-то не так:
1. **Откройте консоль браузера** (F12)
2. **Посмотрите логи** - там будет детальная информация:
   ```
   🔍 Обнаружена сигнатура Pattern Maker: XSPPLAT
   🔍 Парсинг бинарного XSP, размер файла: 123456
   📝 Найденные строки: [...]
   ```
3. **Скопируйте логи** и сообщите о проблеме

## Что смотреть в логах:

### Хорошие признаки ✅:
```
✅ Обнаружен Pattern Maker формат
📝 Найденные строки: ["My Pattern", "Author Name", ...]
🎨 Найдено DMC кодов: 15
📊 Результаты парсинга Pattern Maker:
  Название: My Pattern
  Автор: Author Name
  Размер: 150 x 200
  Цветов: 15
```

### Проблемы ⚠️:
```
⚠️ DMC коды не найдены, добавляю цвета по умолчанию
  → Палитра не была найдена в файле

  Название: Untitled Pattern
  → Название не было извлечено
```

## Анализ формата:

### Для детального анализа:
1. Нажмите **"Анализ формата"**
2. Посмотрите **HEX дамп** первых байтов
3. Проверьте что начинается с `58 53 50 50 4C 41 54` (XSPPLAT)

### Пример правильного HEX:
```
Offset  Hex                                          ASCII
------  -------------------------------------------  ----------------
0000    58 53 50 50 4C 41 54 20 20 00 00 00 00 00   XSPPLAT  .....
0010    ...
```

## Если данные неточные:

### Вариант 1: Экспорт в XML (Рекомендуется)
1. Откройте файл в **Pattern Maker**
2. **File** → **Export** → **XML format**
3. Сохраните как новый файл
4. Загрузите XML версию

### Вариант 2: Используйте как есть
- Данные могут быть неточными, но рабочими
- Проверьте название, автора, размеры
- При необходимости исправьте вручную
